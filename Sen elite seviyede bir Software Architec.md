 Tek kişilik bir yazılım şirketi olarak amaç bir defa yaz, her projede kullan. affexaiFactory bir SaaS değil — yeni bir SaaS / müşteri projesi başlatıldığında 5 dakikada hot-start eden bir yazılım fabrikası. Bu repo:

 - AI agent / prompt / workflow altyapısı (Aider, OpenHands, LangGraph, Langfuse, Promptfoo).

 - Mevcut NestJS / NextJS repo'larının kademeli olarak içeri taşınma planı.
 - GitHub Actions → Coolify deploy hattı, Plausible / Uptime Kuma / Langfuse ile observability.

 Çıktı: yeni MVP'yi 1 günde (eskiden 1 hafta) ayağa kaldıran, AI agent'ların kendi başına PR açabildiği, tek kişi ile 5+ ürün operate edebilecek bir mimari.

 7. Mevcut Projelerin Migration Stratejisi

 Karar: Strangler-fig (kademeli) migration, big-bang DEĞİL.

 Faz 0 — Audit (tools/migrate-repo audit ../old-project):
 - Graphify ile dependency graph çıkar.
 - Eski projedeki utility / config / type'ları packages/*-core ile diff'le.
 - migration-report.md üretir: hangi dosyalar auth-core'a, hangileri db-core'a taşınabilir.

 Faz 1 — Parazit modu (eski repo durur, affexaiFactory paketleri tüketir):
 1. Eski repo'ya .npmrc ekle (Verdaccio registry).
 2. Düşük riskli paket: shared-config (tsconfig, biome) → npm i -D @affex/shared-config. Eski config sil.
 3. Sonra shared-types → eski tipleri Zod schema'ya port, factory ile generate.
 4. observability-core → eski logger drop, pino+Langfuse yeni.

 Faz 2 — Modül migration (her sprint 1 modül):
 - auth-core → eski auth controller'ı Nest module ile değiştir.
 - billing-core → Stripe wrapper'ı drop, core kullan.
 - Test: contract test (testing-utils) ile eski API surface backward-compatible.

 Faz 3 — Repo absorpsiyonu (opsiyonel):
 - Eski repo git subtree add ile apps/<old-name>/ altına taşınır (history korunur).
 - Eski repo arşivlenir (read-only).

 tools/migrate-repo komutları:
 pnpm migrate audit <path>         # rapor üret
 pnpm migrate adopt <path> --pkg shared-config    # belirli bir paketi adopte et
 pnpm migrate absorb <path>        # subtree olarak içeri al

 Komut:
 pnpm create @affex/app
 # interactive prompts:
 #   - name: ticket-saas
 #   - template: nest-saas | next-app | nest-microservice | ai-agent
 #   - features: [auth, billing, ai, notifications, queue, ui-kit] (multiselect)

 tools/create-app implementasyonu (Plop.js + custom logic):

 1. templates/<choice>/ kopyalanır.
 2. Token'lar değiştirilir: __APP_NAME__, __APP_PORT__, __DB_NAME__.
 3. Seçilen feature'lara göre package.json'a @affex/<feature>-core deps eklenir.
 4. apps/<name>/ altına yazılır.
 5. pnpm install + turbo run build --filter=<name> ile sanity check.
 6. Deploy seçildiyse infra/coolify/ altına stub project tanımı düşer.
 7. AI agent config seçildiyse apps/<name>/AGENTS.md ve apps/<name>/.aider.conf.yml enjekte edilir.
 8. Git commit: feat: scaffold <name> from <template>.

 Sürdürülebilirlik: apps/_starter-* her CI'da build/test edilir. Template ile starter arasında drift olursa CI fail. Generator'ın çıktısı E2E test (Playwright ile yeni app açıp pnpm dev çalıştırır) — tools/create-app/__tests__/.
